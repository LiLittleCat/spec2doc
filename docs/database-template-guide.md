# 数据库设计文档 — 自定义模板教程

## 快速开始

本程序可以读取数据库结构信息，结合你的 `.docx` 模板自动生成数据库设计文档。你只需在 Word 中按公司要求设计好样式，在需要动态内容的位置写入占位符，程序就会自动填充数据并输出成品文档。

**核心流程：**

1. 用 Word 打开模板文件，按项目要求调整排版样式
2. 在需要动态内容的位置写入 `{占位符}`
3. 程序读取数据库结构 + 模板 → 输出成品文档

---

## 占位符语法

模板支持三种语法：**简单变量**、**循环块** 和 **条件块**。

### 1. 简单变量

用 `{variableName}` 表示，生成时直接替换为对应的值。

```
{database}     →  my_app_db
{updateDate}   →  2026-02-14
```

### 2. 循环块

用 `{#listName}...{/listName}` 表示，包裹的内容会对列表中的每一项重复生成。

```
{#tables}
表名：{name}
备注：{comment}
{/tables}
```

如果数据库有 10 张表，上面的内容会重复 10 次，每次填入不同表的信息。

### 3. 条件块

用 `{#conditionName}...{/conditionName}` 表示。当条件为 `true` 时，内部内容才会出现在最终文档中。

```
{#hasIndexes}
这一段只在该表有索引时才显示
{/hasIndexes}
```

## 可用变量一览

### 文档级别变量

| 占位符 | 说明 | 示例值 |
|--------|------|--------|
| `{database}` | 数据库名称 | my_app_db |
| `{updateDate}` | 文档更新日期 | 2026-02-14 |

### 数据表级别（在 `{#tables}...{/tables}` 内使用）

| 占位符 | 说明 | 示例值 |
|--------|------|--------|
| `{name}` | 表名 | t_user |
| `{comment}` | 表注释/说明 | 用户信息表 |

### 条件变量（在 `{#tables}` 内使用）

| 条件占位符 | 何时为 true |
|------------|-------------|
| `{#hasIndexes}` | 该表存在索引时 |

### 字段级别（在 `{#columns}...{/columns}` 内使用）

| 占位符 | 说明 | 示例值 |
|--------|------|--------|
| `{colName}` | 字段名 | user_id |
| `{colType}` | 数据类型 | varchar(64) |
| `{colNullable}` | 是否允许空 | 是 / 否 |
| `{colIsPrimary}` | 是否主键 | 是 / 否 |
| `{colIsForeign}` | 是否外键 | 是 / 否 |
| `{colDefault}` | 默认值 | NULL / 0 / CURRENT_TIMESTAMP |
| `{colComment}` | 字段说明 | 用户唯一标识 |

### 索引级别（在 `{#indexes}...{/indexes}` 内使用）

| 占位符 | 说明 | 示例值 |
|--------|------|--------|
| `{idxName}` | 索引名称 | idx_user_email |
| `{idxType}` | 索引类型 | UNIQUE / NORMAL / FULLTEXT |
| `{idxColumns}` | 索引包含的字段 | email, status |

## 数据层级结构

模板中的变量存在嵌套层级关系，完整结构如下：

```
文档顶层
├── {database}
├── {updateDate}
│
└── {#tables}                    ← 遍历每张数据表
     ├── {name}
     ├── {comment}
     │
     ├── {#columns}              ← 遍历该表的每个字段
     │    ├── {colName}
     │    ├── {colType}
     │    ├── {colNullable}
     │    ├── {colIsPrimary}
     │    ├── {colIsForeign}
     │    ├── {colDefault}
     │    └── {colComment}
     │
     └── {#hasIndexes}           ← 条件：该表有索引时显示
          └── {#indexes}         ← 遍历每个索引
               ├── {idxName}
               ├── {idxType}
               └── {idxColumns}
```

## 自定义模板实战指南

### 场景：用公司提供的样式模板

**步骤 1：保留样式，清空内容**

打开公司的 Word 模板，保留页眉/页脚、字体、颜色、表格样式等，清空示例正文。

**步骤 2：按层级填入占位符**

以下是一个示意（在 Word 里按你的样式编写）：

---

> **（以下为模板内容示意）**
>
> 标题 → 数据库设计文档
>
> 表格：
> | 项目 | 值 |
> |------|----|
> | 数据库名称 | `{database}` |
> | 更新日期 | `{updateDate}` |
>
> `{#tables}`
>
> 二级标题 → `{name}` `{comment}`
>
> **字段列表**
>
> | 字段名 | 数据类型 | 允许空 | 主键 | 外键 | 默认值 | 字段说明 |
> |--------|----------|--------|------|------|--------|----------|
> | `{#columns}{colName}` | `{colType}` | `{colNullable}` | `{colIsPrimary}` | `{colIsForeign}` | `{colDefault}` | `{colComment}{/columns}` |
>
> `{#hasIndexes}`
>
> **索引信息**
>
> | 索引名 | 索引类型 | 索引字段 |
> |--------|----------|----------|
> | `{#indexes}{idxName}` | `{idxType}` | `{idxColumns}{/indexes}` |
>
> `{/hasIndexes}`
>
> `{/tables}`

---

### 步骤 3：表格行循环的写法

与接口文档模板一样，**开始标记写在行首列，结束标记写在行末列**：

```
| {#columns}{colName} | {colType} | ... | {colComment}{/columns} |
```

程序会根据字段数量自动重复生成多行。

### 步骤 4：保存并使用

编辑完成后保存为 `.docx`，传给程序即可生成。

## 常见问题

**Q：不需要索引信息，可以去掉吗？**

可以。直接删除 `{#hasIndexes}...{/hasIndexes}` 整块即可。

**Q：想增加字段列（如"是否自增"），怎么办？**

这取决于程序是否提供了对应的变量。如果程序支持 `{colAutoIncrement}` 之类的变量，直接在表格中加一列写上即可。如果不支持，需要先在程序端扩展数据提取逻辑。

**Q：想去掉某些列（如不需要"外键"列），可以吗？**

可以。字段表格中不写 `{colIsForeign}` 那一列就行，其他列照常使用。

**Q：表名和注释可以分开写吗？**

可以。默认模板把 `{name}` 和 `{comment}` 写在同一行作为标题，你完全可以分开处理，比如把 `{name}` 作为标题，`{comment}` 放在正文段落中。

**Q：占位符格式被 Word 拆开了怎么办？**

确保占位符文本格式一致。例如 `{colName}` 不要一半加粗一半不加粗，否则 Word 会将其拆成多个 XML 节点导致程序无法识别。建议先用纯文本写好占位符，再统一选中设置格式。
